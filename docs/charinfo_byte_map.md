Differential analysis of 5 retail CharInfo captures.
Chars: Krafteo, Norman Gates, Hannibal Lecture, Father Augusto, Oda Daramitz

Each section is shown as a side-by-side hex matrix per byte position. Bytes that are identical across all chars are CONSTANT (structural). Bytes that differ are candidate field data.

## Negative findings

**Soullight is NOT in CharInfo at any encoding.** Tested across full common-length region:
- Direct: u8, s8, u16le, s16le → no offset hits ≥3/5 chars.
- Transformed: byte=SL+K, byte=SL XOR K, byte&0x7F=SL&0x7F, u16le=SL+K, low/high nibble = SL nibble → all 6 brute-force searches return zero hits across all 5 chars.

Conclusion: SL travels via TCP, OR is computed locally from another field (e.g., faction sympathy sum from S12). Capture a retail SL-change event to crack it.
